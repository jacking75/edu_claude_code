# Claude Code - Git Worktree

**핵심 개념:**
Claude Code에 Git Worktree 지원이 추가되어, 여러 에이전트가 **하나의 레포지토리에서 서로 충돌 없이 독립적으로 동시 작업**할 수 있게 되었습니다. 기존 데스크탑 앱에서 지원되던 기능이 이제 CLI에도 적용되었습니다.

---

## 병렬 에이전트 활용 5가지 방법

**1. CLI에서 격리된 환경 실행**
`claude --worktree` 옵션 사용으로 간단히 적용. 여러 세션을 동시에 실행해도 코드 충돌 없음. `--tmux` 플래그를 함께 사용하면 각 세션을 별도 Tmux 창에서 관리 가능.

**2. 데스크탑 앱에서 간편 사용**
Claude 데스크탑 앱 'Code' 탭에서 `worktree mode` 체크박스만 활성화하면 CLI 없이도 동일한 격리 환경 이용 가능.

**3. 서브 에이전트 병렬 처리**
서브 에이전트에도 Worktree 격리 지원. 대규모 리팩토링·마이그레이션 작업 시 여러 에이전트가 병렬로 처리 가능. CLI, 데스크탑, IDE 확장, 웹, 모바일 등 전 플랫폼 지원.

**4. 커스텀 에이전트 전용 격리 설정**
커스텀 에이전트의 설정 파일(frontmatter)에 `isolation: worktree` 한 줄 추가 시, 해당 에이전트 호출마다 자동으로 전용 Worktree 생성.

**5. Git 외 버전 관리 시스템 지원**
Mercurial, Perforce, SVN 등 非 Git 환경에서도 Worktree hooks 정의를 통해 동일한 격리 혜택 적용 가능.
