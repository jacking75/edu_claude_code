# Claude Code 실용 소기술 11선

[출처](https://qiita.com/mu7kata/items/c325740e4710120b0963 )  

  
## **1. `AskUserQuestion` 툴로 선택지 입력 효율화:**
`AskUserQuestion`은 대화 중 클릭 방식의 선택지를 제시해주는 Claude Code 내장 툴이다. 이를 사용하면 선택을 요구받을 때마다 텍스트를 직접 입력하지 않아도 된다.

CLAUDE.md 또는 커스텀 슬래시 커맨드에 아래 한 줄을 추가하면 활성화된다.

```
유저에게 선택·판단을 요구하는 경우에는 AskUserQuestion 툴을 사용할 것
```


## **2. 에디터에서 서제스트 입력 (탭 자동완성):**
입력 에디터 안에서 프롬프트를 제안해주는 기능이다. 제안된 내용은 `Tab` 키를 누르면 채택된다.

**설정 방법:** `/config` 커맨드에서 `Prompt suggestions`를 `true`로 설정한다.

  
## **3. Markdown 파일 생성으로 출력 결과 개선:**
응답을 Markdown 파일로 출력하도록 지시하면 제목·구조화가 유도되어 정보량이 많은 답변이 나오기 쉬워진다.

```
/review 리뷰 결과는 md 파일에 출력해주세요
```

특히 리뷰나 설계 지적처럼 제목·목록·코드 블록을 많이 사용하는 답변에 효과적이다. 임시 md 파일이 쌓이기 쉬우므로, 로컬에 `tmp` 디렉터리를 만들어 git 관리 대상에서 제외해두는 것을 권장한다.

```
├── .claude
│   └── tmp  # 저장되는 md 파일은 git 관리 외로 설정
```


## **4. `Ctrl + G`로 Plan 모드 결과를 외부 에디터에서 편집:**
Plan 모드(`Shift + Tab`)에서 계획 생성이 완료된 타이밍에 `Ctrl + G`를 누르면 Vim 등의 외부 에디터로 편집할 수 있다. 생성 결과를 파기하고 다시 계획을 만들게 하면 시간이 걸리므로, 수동으로 직접 수정할 수 있는 것이 편리하다.


## **5. `Ctrl + S`로 프롬프트 임시 저장 (Stash):**
에디터에 입력한 프롬프트를 임시 저장할 수 있다. 저장한 프롬프트를 표시하려면 다시 `Ctrl + S`를 누르거나, 다른 프롬프트를 전송하면 표시된다. 복붙 운용을 하지 않아도 되므로 편리하다.


## **6. 커스텀 슬래시 커맨드를 서브 디렉터리별로 관리:**
커스텀 슬래시 커맨드는 디렉터리 구조에 의한 네임스페이스를 지원한다. 서브 디렉터리로 분류하면 구조적으로 정리할 수 있다.

```
.claude/commands/
├── dev/
│   ├── code-review.md
│   └── refactor.md
├── test/
│   ├── generate-unit.md
│   └── coverage.md
```

실행 형식은 `/{서브 디렉터리명}:{커맨드 파일명}`이다.

```
/test:generate-unit
```

`/test:`까지 입력하면 하위 커맨드가 서제스트되어 찾기 쉬워진다. 또한 표준 내장 커맨드와 이름이 겹치는 것을 피할 수 있다는 장점도 있다.



## **7. `Verbose output`으로 Claude Code의 동작을 분석:**
`Verbose output`을 `true`로 설정하면 대화 중 발생하는 상세 로그를 활성화할 수 있다. 일반적으로 생략·압축되는 부분(턴별 주고받기나 실행 결과의 상세)을 확인할 수 있다. CLAUDE.md나 Skills 등 커스터마이즈한 프롬프트의 동작을 검증할 때 유용하다.

일시적으로 확인하고 싶은 경우에는 `Ctrl + O`로도 동일하게 상세 로그를 펼칠 수 있다.

**설정 방법:** `/config` 커맨드에서 `Verbose output`을 `true`로 설정한다.

  
## **8. `/add-dir`로 프로젝트 외부 디렉터리를 작업 대상에 포함:**
`/add-dir` 커맨드를 사용하면 Claude Code의 작업 대상에 복수의 디렉터리를 추가할 수 있다. 프론트엔드와 백엔드를 별도 리포지터리로 관리하고 있어도, 프론트 쪽에서 작업하면서 백엔드 구현을 고려한 지시나 리뷰가 가능해진다.

```
/add-dir [대상 디렉터리 경로]
```

예시 — 프론트엔드 프로젝트 루트에서 실행:

```
/add-dir ../backend
```

실행 결과는 `.claude/settings.json`의 `additionalDirectories`에 저장되기 때문에 직접 편집해도 동일하게 설정을 반영할 수 있다.

```json
{
  "permissions": {
    "additionalDirectories": [
      "../backend"
    ]
  }
}
```

## **9. `/statusline`으로 컨텍스트 사용률 모니터링:**

컨텍스트 사용률이 임계값에 달하면 Claude Code가 자동으로 대화 이력을 요약하는 기능이 있어, 의도치 않게 문맥이 손실되는 경우도 있다. `/statusline` 커맨드를 사용하면 터미널 하단에 컨텍스트 사용률 등을 표시하는 정보 바를 설치할 수 있다.

표시 내용을 커스터마이즈하려면 커맨드에 이어서 지시를 추가한다.

```
/statusline 컨텍스트 사용률, 모델명, Git 브랜치를 표시해줘
```

실행하면 Claude Code가 지시를 바탕으로 `statusline.sh` 구축과 `settings.json` 설정을 자동으로 생성한다. 컨텍스트 사용률 외에도 브랜치, 모델, 변경 행 수, 실행 시간 등을 표시하도록 커스터마이즈할 수 있다.


## **10. 셸 커맨드를 직접 실행하는 `bash mode`:**
`bash mode`를 활성화하면 입력 에디터 안에서 셸 커맨드를 실행할 수 있다. 입력 앞에 `!`를 붙이면 된다.

```
!npm run build
```

실행한 커맨드명과 출력이 그대로 컨텍스트에 들어가기 때문에, 커맨드가 에러를 반환하는 경우의 조사나 Claude와의 대화 흐름 속에서 셸 커맨드를 실행하고 싶을 때 유효하다. 빌드나 테스트 실행 커맨드가 실패하는 경우에 특히 유용하다.

  
## **11. Claude Code가 디렉터리를 잃어버리는 문제 해결:**
태스크 실행 중 `cd` 커맨드 등으로 디렉터리 이동이 발생하면 다음 태스크에서도 이동한 위치가 유지되어, Claude가 현재 작업 디렉터리를 잃어버리는 경우가 있다.

`CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR`을 활성화하면 각 Bash 커맨드 실행 후 작업 디렉터리가 프로젝트 기점으로 돌아온다. 불필요한 `cd` 커맨드 실행에 의한 토큰 소비도 억제할 수 있다.

**설정 방법:** `.claude/settings.json`에 아래를 추가한다.

```json
{
  "env": {
    "CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR": "1"
  }
}
```
