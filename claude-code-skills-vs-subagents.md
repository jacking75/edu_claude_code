# **Claude Code: Skills vs SubAgents 핵심 정리**

[출처](https://zenn.dev/tmasuyama1114/articles/claude_code_skills_vs_subagents )  

---

**개요:**  
Claude Code를 사용하다 보면 반드시 맞닥뜨리는 질문이 있다. "Skills와 SubAgents, 뭘 써야 하지?" 이 둘을 잘못 사용하면 컨텍스트(AI의 메모리 용량)를 낭비하거나 기대한 정밀도가 나오지 않는다. 반대로 올바르게 구분해서 사용하면 Claude Code의 진가를 끌어낼 수 있다.

---

**한 줄 결론:**

$$\text{「방법」을 가르치고 싶다면 Skills, 「일」을 맡기고 싶다면 SubAgents}$$


## **Skills (스킬) — "필요할 때만 여는 참고서":**
Skills는 AI가 필요하다고 판단했을 때만 해당 정보를 읽어들이는 기능이다. 기술적으로는 **"컨텍스트의 동적 로딩(Dynamic Loading)"** 이라고 부른다.

만약 `CLAUDE.md`에 모든 정보를 적어두면, 그 정보는 항상 메모리를 점유하게 된다. Skills는 이 문제를 해결한다. 레시피 책처럼, 평소엔 책장에 꽂혀 있다가 카르보나라를 만들고 싶을 때만 이탈리안 요리 책을 꺼내는 것과 같다.

**주요 활용 시나리오:**
- **디자인 시스템**: UI를 만들 때만 "우리 회사 디자인 가이드라인"을 읽어들이게 함
- **특정 라이브러리**: `Shadcn UI`를 쓸 때만 해당 베스트 프랙티스를 읽어들이게 함
- **코딩 규약**: 특정 모듈 작업 시에만 관련 네이밍 규칙, 컨벤션 등을 로딩

Anthropic 공식에서도 디자인 일관성을 유지하기 위해 Skills 활용을 권장하고 있다.


## **SubAgents (서브에이전트) — "조사를 맡기는 별동대":**
SubAgents는 **독립적인 컨텍스트를 가진 별도의 AI 에이전트**다. 메인 Claude와는 별개로 또 다른 Claude를 호출해 특정 작업을 맡기는 구조다.

이 "별동대"는 메인 대화 기록을 모른다. 따라서 독립적으로 작업하며, 아무리 대량의 파일을 읽어도 메인 컨텍스트가 오염되지 않는다. 즉, **메인 Claude는 설계·본질적 작업에 집중**할 수 있다.

프로젝트 매니저(메인 Claude)가 전체 설계를 고민하는 도중 버그를 발견했다고 하자. 수백 줄의 로그를 직접 추적하면 집중력이 흐트러진다. 그때 SubAgent에게 "이 에러 로그의 원인을 찾아서 결과만 알려줘"라고 맡기면 된다. SubAgent가 대량의 토큰을 소비하며 원인을 특정하고, 메인 Claude에게 결과만 보고한다.

**주요 활용 시나리오:**
- **대규모 조사**: 프로젝트 전체 파일에서 특정 클래스나 함수 사용처를 모두 추출
- **문서 읽기**: 새 라이브러리 문서를 통째로 읽히고 기능 사용법 요약 요청
- **구현 검증**: 현재 구현이 올바른지 객관적인 시각으로 테스트 요청


## **비교표:**

| 항목 | Skills | SubAgents |
|---|---|---|
| 이미지 | 참고서, 매뉴얼 | 어시스턴트, 별동대 |
| 컨텍스트 | 메인 대화에 읽혀 들어옴 | **독립** (메인 오염 없음) |
| 토큰 소비 | 사용 시에만 소비 (절약 효과) | 별도 예산으로 소비 (메인 절약) |
| 주요 용도 | 지식 제공, 룰 적용 | 조사, 탐색, 검증 |
| 지속성 | 태스크 종료 시 소멸 | 태스크 종료 시 소멸 |

  

## **판단 플로우차트:**

```
Q1. AI에게 지켜야 할 규칙이나 지식을 주고 싶은가?
    → YES: Skills 사용
       (예: 디자인 룰, 네이밍 규칙, 라이브러리 사용법)

Q2. 대량의 파일을 읽거나 시행착오가 필요한 작업인가?
    → YES: SubAgents 사용
       (예: 버그 조사, 영향 범위 파악, 문서 요약)
```

  
## **핵심 요약:**
Skills와 SubAgents는 역할이 명확히 다르다. Skills는 지식을 효율적으로 제공하고, SubAgents는 지저분한(토큰 소비가 많은) 조사 작업을 독립적으로 처리한다. 이 역할 분담이 잘 이루어지면 개발 효율이 크게 향상된다.
